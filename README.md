## What this RaaS patent portfolio covers

This patent portfolio is structured around companion-robot RaaS use cases where a digital service request becomes a real-world robot behavior. The claims are presented through five practical UX scenes that connect business-method flow, platform orchestration, and physical robot action.

Covered examples include:

- service-triggered exercise support by companion robots;
- user-state-aware timing for exercise encouragement;
- robot-side physical gestures after insufficient user activity;
- indoor-to-outdoor companion mode switching;
- outdoor accompaniment and user monitoring;
- hazard guidance by moving to a leading position;
- shielding behavior between a user and an approaching moving body;
- illumination-based path guidance and warning lights; and
- RaaS transactions connecting service providers, robotics operators, and physical robot actions.

## Why this matters for RaaS platforms

Many robot products treat services as pre-installed robot skills or isolated applications. This portfolio is positioned around a broader RaaS layer: third-party service providers can request outcomes, robotics operators can orchestrate execution, and companion robots can convert those requests into user-facing physical behavior.

That distinction matters because the protected UX is not limited to screen messages or software notifications. The robot can approach the user, time a prompt based on user state, change operating modes when moving outdoors, guide the user away from hazards, physically interpose itself as a guard, and project light to influence movement.

## Strategic relevance

### Robot developers and manufacturers

Turn deployed companion robots into monetizable service endpoints. Patented RaaS control flows can connect robots with health, safety, insurance, and care-service partners, expanding revenue beyond hardware sales or pre-installed robot skills.

### Health, insurance, and safety service providers

Extend digital services into the user's physical world. Companion robots can prompt exercise, accompany users outdoors, guide safer movement, and provide safety-related support around an individual user.

### Cloud robotics, robot API, and RaaS platforms

Build a strategic intermediation layer between service providers and physical robots. The portfolio covers RaaS transactions that translate service requests into autonomous physical robot actions, enabling platform operators to orchestrate services on top of companion robots.

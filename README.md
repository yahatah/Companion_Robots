# Companion_Robots

Landing page for a patent portfolio covering **companion robots**, **personal service robots**, and **Robot-as-a-Service (RaaS)** architectures for physical service delivery.

This repository hosts a static GitHub Pages landing page introducing granted patents available for acquisition. The portfolio focuses on user-facing companion robots that translate third-party service requests and robot-side sensing into physical actions around an individual user.

## What this patent portfolio covers

The portfolio is structured around companion-robot RaaS use cases where digital or service-provider requests become real-world robot behaviors. Covered examples include:

- Service-triggered exercise support by companion robots
- User-state-aware timing for exercise encouragement
- Robot approaches and exercise-prompting gestures after insufficient user activity
- Indoor-to-outdoor companion mode switching
- Outdoor accompaniment and user safety support
- Hazard guidance by moving to a leading position
- Shielding behavior between the user and an approaching moving body
- Illumination-based path guidance and warning lights
- RaaS transactions connecting service providers, robot operators, and physical robot actions

## Strategic relevance

This IP portfolio may be relevant to:

### Robot Developers & Manufacturers

Turn deployed companion robots into monetizable service endpoints. Use patented RaaS control flows to connect robots with health, safety, insurance, and care-service partners — expanding revenue beyond hardware sales or pre-installed robot skills.

### Health, Insurance & Safety Providers

Extend digital services into the user's physical world. Add premium service tiers where companion robots can prompt exercise, accompany users outdoors, guide safer movement, and provide safety-related physical support around the individual user.

### Cloud, Robot API & RaaS Platforms

Build a strategic intermediation layer between service providers and physical robots. Covers key RaaS transactions that translate service requests into autonomous physical robot actions — enabling platform operators to orchestrate multiple services on top of companion robots.
